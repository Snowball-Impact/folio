# FOLIO 프로젝트 컨텍스트

이 문서는 새 대화나 새 컨텍스트에서 FOLIO 작업을 이어갈 때 반복해서 전달해야 하는 주요 지침과 현재 구현 상태를 정리한 것이다.

## 문서 유지 규칙

- 새 주요 지침, 반복 작업 원칙, 중요한 구현 결정, 주의사항이 생기면 이 문서를 자동으로 업데이트한다.
- 사용자가 다음 컨텍스트로 옮기고 싶다고 하거나, 기존 지침을 다시 설명하는 상황이 반복되면 이 문서를 먼저 최신화한다.
- 코드와 문서가 다르면 코드베이스를 확인한 뒤 이 문서를 현재 상태에 맞게 고친다.
- 민감 정보는 이 문서에 기록하지 않는다.

## 프로젝트 방향

FOLIO는 데이터 분석 프로젝트를 포트폴리오 자산으로 등록, 탐색, 공유하는 MVP다.

현재 Week 1~2 핵심 기능은 대부분 구현되었고, UI/UX 마감과 탐색 경험 보강을 진행 중이다. 핵심 메시지는 다음과 같다.

- 발표로 끝나지 않는 프로젝트
- AI 시대에도 휴먼 인사이트는 자산이다.
- 데이터 분석 프로젝트를 포트폴리오로 축적하고 공유하세요.

## 작업 원칙

- 매번 `compileall`이나 `git diff`를 실행하지 않는다.
- 단순 CSS, 문구, 작은 스타일 변경은 과한 검증을 생략한다.
- 파이썬 구조 변경, 복잡한 리팩토링, 에러 가능성이 큰 수정 때만 필요한 파일 단위로 확인한다.
- 사용자가 말한 UI 이름을 코드 클래스명으로 바로 단정하지 않는다.
- 먼저 관련 마크업, Streamlit 래퍼, CSS 적용 범위를 분석한 뒤 수정한다.
- Streamlit 전역 CSS 오염에 특히 주의한다.
- 특정 페이지나 컴포넌트 스타일은 가능한 전용 클래스 기준으로 스코프를 좁힌다.
- 사용자 변경으로 보이는 작업트리 변경은 되돌리지 않는다.

## UX 및 기능 기준

- 카테고리는 사용하지 않는다.
- 프로젝트 탐색은 태그 검색 중심이다.
- 포트폴리오 입력 폼은 여러 필드로 강제 분리하지 않고, 자유 입력 본문을 유지한다.
- PC 화면이 우선이다.
- 모바일 최적화는 하되, PC 레이아웃이 모바일처럼 좁아지면 안 된다.
- 로그인/회원가입은 중앙의 작고 세련된 인증 모듈로 배치한다.
- 인증 화면은 불필요한 히어로/설명/중복 링크를 줄이고, 폼 자체에 집중한다.

## 현재 주요 화면 상태

### 홈

- 홈 히어로 카피:
  - 제목: `AI 시대에도 휴먼 인사이트는 자산이다.`
  - 설명: `데이터 분석 프로젝트를 포트폴리오로 축적하고 공유하세요.`
- 홈 히어로는 헤더와 자연스럽게 이어지는 어두운 네이비 계열이다.
- 홈 히어로는 너무 커지지 않도록 높이, 제목 크기, 텍스트 간격을 줄인 상태다.
- 홈 카드와 둘러보기 카드는 버튼 없이 카드 전체 클릭으로 상세 이동한다.
- 카드 링크의 기본 파란색/밑줄 스타일은 제거한다.

### 네비게이션

- 상단 헤더의 메뉴는 햄버거 버튼 안으로 모았다.
- 햄버거 버튼은 헤더 오른쪽에 유지한다.
- 모바일 폭에서도 햄버거가 아래로 내려가지 않도록 헤더는 row 배치를 유지한다.
- 메뉴 드롭다운은 햄버거 근처에 작게 붙어 열리는 형태다.
- 비로그인 네비게이션에 `회원가입` 메뉴는 노출하지 않는다.
- 단, `?page=Sign Up` 라우트는 유지한다. 로그인 페이지의 회원가입 링크로 접근 가능해야 한다.

### 인증 페이지

- 로그인/회원가입 페이지의 상단 큰 히어로는 제거했다.
- 인증 페이지는 다음 구조다.
  - 투명 중앙 래퍼
  - 어두운 인증 헤드
  - 흰색 폼 카드
- 인증 헤드에는 `F`, `LOGIN`, `SIGN UP` 같은 레이블/배지를 두지 않는다.
- 인증 헤드의 상단 좌우 라운드는 제거되어 있다.
- 회원가입 페이지에서는 다음 요소를 제거했다.
  - `가입 후 받은 인증 메일을 확인한 다음, Login 메뉴에서 로그인하세요.` 안내 박스
  - 하단 `이미 계정이 있다면 로그인하기` 링크
- 회원가입 페이지의 `인증 메일 다시 받기`는 접힌 보조 영역으로 둔다.
- 회원가입 입력 검증은 문제가 있을 때만 오류/경고를 보여준다. 정상 상태마다 초록 성공 메시지를 띄우지 않는다.
- 비밀번호 입력폼은 다른 입력폼과 디자인을 통일한다.
- 비밀번호 눈 아이콘은 입력 래퍼 안에서 중앙 정렬되도록 유지한다.

### Gallery / 상세

- 상세 페이지는 Gallery 메뉴 안에서 `project_id` 쿼리로 렌더링한다.
- 상세 페이지 하단 버튼 텍스트는 `목록으로 돌아가기`다.
- 상세 페이지 대시보드와 첨부파일은 우측 사이드에 배치한다.
- 대시보드 제목은 `대시보드`다. Power BI 문구는 노출하지 않는다.
- 대시보드 iframe은 우측 사이드에서 16:12 비율로 조정되어 있다.
- 둘러보기 카드의 조회/좋아요/태그는 우하단 배치다.

## 데이터 및 백엔드 상태

- DB에서 `projects.category`는 제거했다.
- 코드에서도 category 관련 필터, 폼, 페이로드는 제거했다.
- Supabase JWT expired 문제 대응:
  - public read 중 JWT 만료 시 anon client로 복구하는 로직이 있다.
- 좋아요 수와 좋아요순 정렬은 실제 `likes` DB 데이터를 기반으로 처리한다.
- 상세 화면의 좋아요 버튼 UX 연결은 남은 후보 작업이다.
- `.env`는 민감 정보이므로 열람/노출/커밋에 주의한다.

## CSS 주의사항

- Streamlit의 `stVerticalBlockBorderWrapper` 전역 스타일은 여러 화면을 쉽게 망가뜨린다.
- 인증 페이지 스타일은 `.folio-auth-card-header`, `.folio-auth-form-card`가 있는 래퍼에만 제한한다.
- 과거 문제:
  - `stVerticalBlockBorderWrapper` 전역 스타일 때문에 홈/둘러보기/소개가 같이 작아진 적이 있다.
  - 인증 페이지의 헤드 라운드 문제는 일반 `.folio-hero`가 아니라 `.folio-auth-card-header`에서 온 것이었다.
- 사용자가 말하는 "히어로"가 코드의 `.folio-hero`와 일치하지 않을 수 있다. 실제 화면의 시각적 블록 기준으로 추적한다.
- 한글 문구는 단어가 깨지지 않도록 `word-break: keep-all`, 적절한 `max-width`, 줄간격을 함께 고려한다.

## 로컬 서버

- 이 프로젝트는 Streamlit 앱이다.
- 기본 실행 포트는 `8501`이다.
- 현재 로컬 확인 주소는 보통 `http://localhost:8501`이다.
- 5173은 Vite 기본 포트라 이 프로젝트의 기본값이 아니다.
- `st.set_page_config()`는 엔트리 파일 `app.py` 최상단에서 먼저 호출되도록 정리되어 있다.

## 최근 교훈

- CSS 문제를 섣불리 덮지 말고, 관련 마크업과 실제 적용 클래스부터 확인한다.
- 특히 Streamlit은 사용자가 보는 블록과 코드상 클래스명이 다를 수 있다.
- 캡처나 DOM 구조가 있으면 먼저 원인을 특정하고, 그 다음 최소 수정한다.
