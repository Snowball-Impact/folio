# Design Direction

## Style

- Modern SaaS
- Portfolio Platform
- Clean dashboard layout
- Card-based UI
- Minimal text
- Insight-first content display

## Color

- Navy
- Blue
- White
- Mint accent

## UX Principle

FOLIO의 핵심은 프로젝트를 예쁘게 전시하는 것이 아니라,  
프로젝트에 담긴 문제 정의, 분석 과정, 인사이트를 빠르게 탐색할 수 있게 하는 것이다.

# 9. About

## Purpose

FOLIO의 배경과 비전을 설명한다.

## Layout

```text
┌──────────────────────────────────────────────┐
│ FOLIO 소개                                    │
└──────────────────────────────────────────────┘

발표로 끝나지 않는 프로젝트

FOLIO는 데이터 분석 프로젝트를 포트폴리오 자산으로 축적하고 공유하는 플랫폼입니다.

## 왜 만들었나요?

교육 현장에서 생산된 프로젝트가 발표 이후 사라지는 문제를 해결하기 위해 만들었습니다.

## 핵심 철학

AI 시대에도 휴먼 인사이트는 자산입니다.

## 우리가 검증하는 것

1. 사람들은 자신의 프로젝트를 공개할 의향이 있는가?
2. 사람들은 다른 사람의 프로젝트를 탐색할 의향이 있는가?
```

# Interaction Rules

## View Count

- Portfolio Detail 페이지에 진입할 때 조회수가 증가한다.
- 비회원도 조회수 증가 대상이다.

## Like

- 로그인 사용자만 좋아요를 누를 수 있다.
- 한 사용자는 하나의 프로젝트에 한 번만 좋아요를 누를 수 있다.
- 좋아요 취소는 MVP에서 선택 기능으로 둔다.

## Project Visibility

- 등록한 프로젝트는 즉시 공개된다.
- 관리자 승인 과정은 없다.
- 관리자는 부적절한 콘텐츠를 사후 삭제할 수 있다.

# Core UX Principle

FOLIO는 프로젝트를 단순히 저장하는 공간이 아니다.

사용자가 다음 질문에 빠르게 답할 수 있어야 한다.

1. 이 프로젝트는 어떤 문제를 다루는가?
2. 어떤 데이터를 사용했는가?
3. 어떤 분석 과정을 거쳤는가?
4. 어떤 인사이트를 얻었는가?
5. 작성자는 어떤 역량을 보여주는가?
